# Story 1.1 — 数据库表结构创建

> **Epic**: data-engine (财报数据引擎) | **优先级**: P0 | **阶段**: Phase 1 MVP
> **依赖**: 无 | **预估工时**: 3d

---

## 一、PRD（产品需求文档）

### 1.1 背景

SmartReport 需要统一的数据底座支撑所有分析模块。需创建完整的 MySQL 数据库表结构，涵盖公司信息、财报数据、指标定义、规则引擎、用户交互等全部 17 张表。

### 1.2 需求描述

作为系统，我需要统一的数据库表结构，以支撑所有分析模块的数据读写。

### 1.3 验收标准

| # | 验收条件 | 验证方式 |
|---|---------|---------|
| AC1 | MySQL 中成功创建全部 17 张表，字段类型与 ER 图一致 | `SHOW TABLES` 验证 |
| AC2 | 所有外键约束、索引正确建立 | `SHOW INDEX FROM <table>` |
| AC3 | 种子数据（公司、财报、行业均值）可正常插入并查询 | SELECT 验证 |
| AC4 | Spring Boot 启动时 Hibernate 校验表映射无报错 | 启动日志 |
| AC5 | `SmartReportApplicationTests` 数据源连接测试通过 | `mvn test` |

### 1.4 UI/交互说明

无前端 UI。本 Story 为纯后端基础设施。

## 二、开发方案

### 2.1 技术方案

| 技术点 | 选型 | 说明 |
|--------|------|------|
| 数据库 | MySQL 8.0 | 关系型数据库，适合结构化财报数据 |
| ORM | Spring Data JPA + Hibernate | 自动建表 + 校验模式（validate） |
| 连接池 | HikariCP | Spring Boot 默认，高性能 |
| 索引策略 | 复合索引 | `(company_code, report_year)` 等高频查询路径 |

| 种子数据 | SQL INSERT 脚本 | 预置沪深 300 公司 + 5 年模拟财报 |

### 2.4 实施步骤

**步骤 1：编写 DDL 脚本 `01-init.sql`**
- 按 ER 图定义 17 张表结构
- 设置字符集 `utf8mb4`
- 添加复合索引：`idx_company_year(company_code, report_year)`
- 添加外键约束（CASCADE/SET NULL 按业务需求）
- 添加 `mq_task_records` 表（任务追踪）

**步骤 2：编写种子数据脚本**

`02-seed-company.sql`:
- 预置 300+ A股公司（覆盖白酒、银行、新能源、医药等行业）
- 包含 code, name, short_name, industry, market, listing_date

`03-seed-financial.sql`:
- 为热门公司预置 5 年（2020-2024）财报数据
- 包含 revenue, profit, grossMargin, netMargin, debtRatio, cashFlow, ROE
- 填充 `indicator_definitions` 表（30+ 指标定义 + 白话解释）
- 填充 `industry_averages` 表（按行业 + 年份 + 指标）
- 填充 `highlight_rules` 和 `risk_rules` 规则配置

**步骤 3：配置 Spring Boot 数据源**

`application.yml` 关键配置：
```yaml
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/smartreport?useUnicode=true&characterEncoding=utf8mb4
    username: ${MYSQL_USER:root}
    password: ${MYSQL_PASSWORD:root123}
    driver-class-name: com.mysql.cj.jdbc.Driver
    hikari:
      maximum-pool-size: 20
      minimum-idle: 5
  jpa:
    hibernate:
      ddl-auto: validate  # 仅校验，不自动建表
    show-sql: false
    properties:
      hibernate:
        dialect: org.hibernate.dialect.MySQLDialect
        format_sql: true
```

**步骤 4：创建 JPA Entity 类**

为每张表创建对应的 Entity 类，使用 Lombok `@Data`/`@Entity`/`@Table` 注解，映射字段类型：
- Java `Long` → MySQL `bigint`
- Java `String` → MySQL `varchar`
- Java `BigDecimal` → MySQL `decimal(20,2)`（财务数据精度要求）
- Java `LocalDateTime` → MySQL `datetime`
- JSON 字段使用 `@Column(columnDefinition = "json")` + `@Convert` 或 `JacksonJsonType`

**步骤 5：创建 Repository 接口**

每个 Entity 对应一个 `@Repository` 接口，继承 `JpaRepository<Entity, Long>`。

**步骤 6：编写基础测试**

`SmartReportApplicationTests.java`：
- `testDataSourceConnection()`：验证数据源连接
- `testCompanyTableExists()`：查询 companies 表
- `testSeedDataLoaded()`：验证种子数据条数

### 2.6 风险与注意事项

- ⚠️ `ddl-auto: validate` 模式下，Entity 字段必须与表结构完全一致
- ⚠️ `decimal` 精度设为 `(20,2)` 足够覆盖千亿级营收
- ⚠️ `utf8mb4` 字符集支持 emoji 存储
- ⚠️ Docker volume 挂载 init.sql 仅在首次启动时执行，重复启动不会覆盖
- ⚠️ 种子数据使用 `INSERT IGNORE` 避免重复插入
