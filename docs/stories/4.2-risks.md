# Story 4.2 — 风险识别与展示

> **Epic**: risk-highlight (风险与亮点分析) | **优先级**: P1 | **阶段**: Phase 2 核心闭环
> **依赖**: Story 4.1 (可并行) | **预估工时**: 1.5d

---

## 一、PRD（产品需求文档）

### 1.1 背景

投资决策需要全面了解风险。系统应自动识别财报中潜在的风险信号，帮助用户理性看待公司。

### 1.2 需求描述

作为用户，我希望能看到系统自动识别出的潜在风险项，以红色卡片形式展示。

### 1.3 验收标准

| # | 验收条件 |
|---|---------|
| AC1 | Dashboard 中渲染风险识别区域，含 2-4 张风险卡片 |
| AC2 | 每张卡片红色左边框 + emoji icon + 标题 + 描述 |
| AC3 | 描述文案自动填充公司实际数值 |
| AC4 | 不同公司数据触发不同的风险项 |
| AC5 | 风险卡片与亮点卡片在同一区块左右或上下排列 |

### 1.4 UI 设计

```
┌─────────────────────────────────────────────┐
│  ⚠️ 风险提示                                │
├─────────────────────────────────────────────┤
│ ┃📉 利润增速放缓                            │
│ ┃近两年归母净利润增速从高位回落至 15% 左右， │
│ ┃需关注未来增长持续性。                     │
├─────────────────────────────────────────────┤
│ ┃🏛️ 政策监管风险                            │
│ ┃白酒行业面临消费税调整等政策不确定性，可能  │
│ ┃影响行业整体利润率。                       │
└─────────────────────────────────────────────┘
```

## 二、开发方案

### 2.1 技术方案

与 Story 4.1 共享 `ConditionEvaluator`，仅规则表不同。架构完全一致。

### 2.2 规则设计

| rule_key | 条件 | 风险模板 |
|----------|------|---------|
| `profit_slowdown` | 净利润同比连续2年下降 | 利润增速放缓：近两年归母净利润增速从{yoy_prev}%回落至{yoy}% |
| `policy_risk` | 行业∈[白酒,地产,教培] | 政策监管风险：{industry}行业面临政策调整不确定性 |
| `high_debt` | 资产负债率 > 70% | 高负债风险：资产负债率达{value}%，远超行业均值{avg}% |
| `cashflow_weak` | 经营现金流 < 净利润×0.5 | 现金流薄弱：经营现金流仅为净利润的{ratio}% |
| `revenue_decline` | 营收同比 < 0 | 营收下滑：营收同比下降{abs_yoy}% |

### 2.3 关键文件

| 文件 | 说明 |
|------|------|
| `backend/.../service/RiskService.java` | 风险规则引擎 |
| `backend/.../controller/AnalysisController.java` | GET /api/v1/analysis/{code}/risks |
| `devops/mysql/sql/03-seed-financial.sql` | risk_rules 种子数据 |
| `frontend/src/components/RiskCards.vue` | 风险卡片组件 |

### 2.4 实施步骤

**步骤 1：种子规则数据**

```sql
INSERT INTO risk_rules (rule_key, indicator_key, condition_expr, title, desc_template, priority, enabled) VALUES
('profit_slowdown', 'profit', 'profit_yoy < 20 AND profit_yoy_prev < 25',
 '利润增速放缓', '近两年归母净利润增速从 {profit_yoy_prev}% 回落至 {profit_yoy}% 左右，需关注未来增长的可持续性。', 1, 1),
('policy_risk', null, 'industry IN (白酒,房地产,教培,互联网)',
 '政策监管风险', '{industry}行业面临政策调整的不确定性，可能影响公司经营环境和利润率。', 2, 1),
('high_debt', 'debtRatio', 'debtRatio > 70',
 '高负债风险', '资产负债率高达 {debtRatio}%，远超行业均值 {industryAvgDebtRatio}%，财务杠杆过高，抗风险能力弱。', 3, 1),
('cashflow_weak', 'cashFlow', 'cashFlow < profit * 0.5',
 '现金流薄弱', '经营现金流 {cashFlow} 亿仅为净利润 {profit} 亿的 {ratio}%，利润回收质量偏低。', 4, 1);
```

**步骤 2：RiskService 实现**

与 HighlightService 架构完全一致，仅替换为 `RiskRuleRepository` 和 `RiskItem` DTO。
