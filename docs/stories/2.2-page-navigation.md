# Story 2.2 — 页面间参数传递与加载衔接

> **Epic**: search-load (公司搜索与加载) | **优先级**: P0 | **阶段**: Phase 1 MVP
> **依赖**: Story 2.1 | **预估工时**: 1.5d

---

## 一、PRD（产品需求文档）

### 1.1 背景

用户从搜索页跳转到主控台时，系统需自动加载对应公司数据。MVP 阶段通过 URL 参数传递公司代码，替代 proto 中的 `sessionStorage` 方案。

### 1.2 需求描述

作为用户，从搜索入口页跳转到主控台时，系统应自动加载对应公司数据，并在加载过程中展示骨架屏。

### 1.3 验收标准

| # | 验收条件 |
|---|---------|
| AC1 | 搜索页选择公司 → 路由跳转 `/dashboard/600519` |
| AC2 | Dashboard 页面从 `$route.params.code` 读取公司代码 |
| AC3 | 页面自动调用 `GET /api/v1/reports/{code}/latest` 加载数据 |
| AC4 | 数据加载期间显示骨架屏（Skeleton），非空白闪烁 |
| AC5 | 数据加载完成后渲染 KPI 卡片等模块 |
| AC6 | 加载失败时显示错误提示 + 重试按钮 |

### 1.4 URL 设计

| 路由 | 组件 | 说明 |
|------|------|------|
| `/search` | SearchView | 搜索入口页 |
| `/dashboard/:code` | DashboardView | 主控台（code=股票代码） |
| `/dashboard/:code?tab=predict` | DashboardView | 直达预测模块 |

## 二、开发方案

### 2.1 技术方案

| 技术点 | 选型 | 说明 |
|--------|------|------|
| 路由 | Vue Router 4 | `createRouter` + `createWebHistory` |
| 参数传递 | URL params (`:code`) | 可书签、可分享、可前进后退 |

| 加载态 | SkeletonCard 组件 | CSS 脉冲动画占位 |

### 2.3 实施步骤

**步骤 1：配置路由**

```typescript
// router/index.ts
const routes = [
  { path: '/', redirect: '/search' },
  { path: '/search', name: 'Search', component: () => import('@/views/SearchView.vue') },
  { path: '/dashboard/:code', name: 'Dashboard', component: () => import('@/views/DashboardView.vue'), props: true },
  { path: '/:pathMatch(.*)*', name: 'NotFound', component: () => import('@/views/NotFoundView.vue') },
]
```

**步骤 2：实现 useDashboard composable**

```typescript
// composables/useDashboard.ts
export function useDashboard(companyCode: Ref<string>) {
  const dashboardStore = useDashboardStore()
  const loading = ref(true)
  const error = ref<string | null>(null)

  async function loadCompanyData() {
    loading.value = true
    error.value = null
    try {
      const [latestRes, kpiRes] = await Promise.all([
        getReportLatest(companyCode.value),
        getReportKpi(companyCode.value)
      ])
      dashboardStore.setLatestReport(latestRes.data.data)
      dashboardStore.setKpis(kpiRes.data.data)
    } catch (e) {
      error.value = '加载失败，请稍后重试'
    } finally {
      loading.value = false
    }
  }

  // 路由参数变化时自动重新加载
  watch(companyCode, () => loadCompanyData(), { immediate: true })

  return { loading, error, loadCompanyData, ...toRefs(dashboardStore) }
}
```

**步骤 4：实现后端 `/latest` 端点**

```java
@GetMapping("/{companyCode}/latest")
public ApiResponse<LatestReportResponse> getLatestReport(@PathVariable String companyCode) {
```
