# Story 9.2 — 历史记录云同步

> **Epic**: auth (用户系统) | **优先级**: P3 | **阶段**: Phase 4 商业化
> **依赖**: Story 9.1 | **预估工时**: 1.5d

---

## 一、PRD（产品需求文档）

### 1.1 背景

Story 2.3 的分析历史仅存储在 localStorage 中，无法跨设备同步。引入用户系统后，历史记录应与用户 ID 关联，存储在后端数据库，支持多设备同步。

### 1.2 需求描述

作为用户，我的分析历史可以在多设备间同步，换设备登录后仍能看到之前的分析记录。

### 1.3 验收标准

| # | 验收条件 |
|---|---------|
| AC1 | 登录后分析公司 → 历史写入后端 `user_search_history` 表 |
| AC2 | 换设备登录 → 历史从后端同步显示 |
| AC3 | 未登录 → 降级为 localStorage 存储 |
| AC4 | 历史 API（GET/POST/DELETE）需 JWT 认证 |
| AC5 | 登录后首次加载 → 清空 localStorage，改用后端数据 |
| AC6 | 最多保留 50 条（后端），自动淘汰最旧记录 |

### 1.4 API 清单

| 方法 | 路径 | 认证 | 说明 |
|------|------|------|------|
| GET | `/api/v1/history` | 🔒 | 获取分析历史（最近 50 条） |
| POST | `/api/v1/history` | 🔒 | 记录分析历史 |
| DELETE | `/api/v1/history/{id}` | 🔒 | 删除单条历史 |
| DELETE | `/api/v1/history` | 🔒 | 清空全部历史 |

## 二、开发方案

### 2.1 技术方案

| 技术点 | 选型 | 说明 |
|--------|------|------|
| 渐进式迁移 | localStorage + API 双写 | 平滑过渡，不丢失旧数据 |
| 降级策略 | 未登录 → localStorage | 向后兼容 |
| 数据合并 | 首次登录 → localStorage 数据上传 | 不丢失匿名期间的历史 |

**步骤 1：HistoryController**

```java
@RestController
@RequestMapping("/api/v1/history")
public class HistoryController {

    @GetMapping
    public ApiResponse<List<HistoryItem>> getHistory(@AuthenticationPrincipal Long userId) {
        return ApiResponse.success(historyService.getHistory(userId));
    }

    @PostMapping
    public ApiResponse<Void> addHistory(@AuthenticationPrincipal Long userId,
                                         @RequestBody AddHistoryRequest req) {
        historyService.addHistory(userId, req.getCompanyCode());
        return ApiResponse.success(null);
    }

    @DeleteMapping("/{id}")
    public ApiResponse<Void> deleteHistory(@AuthenticationPrincipal Long userId,
                                            @PathVariable Long id) {
        historyService.deleteHistory(userId, id);
        return ApiResponse.success(null);
    }

    @DeleteMapping
    public ApiResponse<Void> clearHistory(@AuthenticationPrincipal Long userId) {
        historyService.clearHistory(userId);
        return ApiResponse.success(null);
    }
}
```

**步骤 2：HistoryService**

```java
@Service
public class HistoryService {
    private static final int MAX_ITEMS = 50;

    public void addHistory(Long userId, String companyCode) {
        // 去重：删除同公司旧记录
        historyRepo.deleteByUserIdAndCompanyCode(userId, companyCode);

        // 新增
        historyRepo.save(UserSearchHistory.builder()
            .userId(userId).companyCode(companyCode)
            .searchedAt(LocalDateTime.now()).build());

        // 保持最多 50 条
        long count = historyRepo.countByUserId(userId);
        if (count > MAX_ITEMS) {
            historyRepo.deleteOldestByUserId(userId, count - MAX_ITEMS);
        }
    }

    public List<HistoryItem> getHistory(Long userId) {
        return historyRepo.findByUserIdOrderBySearchedAtDesc(userId, PageRequest.of(0, MAX_ITEMS))
            .stream().map(this::toHistoryItem).toList();
    }

    private HistoryItem toHistoryItem(UserSearchHistory h) {
        Company company = companyRepo.findByCode(h.getCompanyCode());
        return HistoryItem.builder()
            .id(h.getId()).code(h.getCompanyCode())
            .name(company.getName()).searchedAt(h.getSearchedAt()).build();
    }
}
```

**步骤 3：改造 useHistory.ts**

```typescript
export function useHistory() {
  const authStore = useAuthStore()
  const items = ref<AnalysisHistoryItem[]>([])
  const isSyncing = ref(false)

  // 初始化
  onMounted(async () => {
    if (authStore.isLoggedIn) {
      // 登录用户：从后端加载 + 上传 localStorage 旧数据
      await syncFromServer()
    } else {
      // 匿名用户：从 localStorage 加载
      loadFromLocal()
    }
  })

  async function add(company: { code: string; name: string }) {
    // 本地先更新（乐观更新）
    items.value = items.value.filter(i => i.code !== company.code)
    items.value.unshift({ ...company, timestamp: Date.now() })
    saveToLocal()

    // 登录用户同步到后端
    if (authStore.isLoggedIn) {
      try {
        await historyApi.addHistory(company.code)
      } catch { /* 静默失败，下次同步时补齐 */ }
    }
  }

  async function syncFromServer() {
    isSyncing.value = true
    try {
      // 1. 上传 localStorage 中的旧数据
      const localItems = loadFromLocal()
      for (const item of localItems) {
        await historyApi.addHistory(item.code).catch(() => {})
      }
      // 2. 从后端拉取完整列表
      const res = await historyApi.getHistory()
      items.value = res.data.data.map(i => ({
        code: i.code, name: i.name, timestamp: new Date(i.searchedAt).getTime()
      }))
      // 3. 清空 localStorage（以后用后端数据）
      localStorage.removeItem(STORAGE_KEY)
    } finally {
      isSyncing.value = false
    }
  }

  return { items, add, remove, clear, isSyncing }
}
```
