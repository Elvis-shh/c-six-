# Story 10.2 — 监控与日志

> **Epic**: infra (系统基建) | **优先级**: P3 | **阶段**: Phase 4 商业化
> **依赖**: Story 10.1 | **预估工时**: 1.5d

## 一、PRD（产品需求文档）

### 1.1 背景

系统上线后需要可观测性：健康检查、API 限流、异常追踪、请求日志。确保问题可快速定位和恢复。

### 1.2 需求描述

作为团队，我们需要可观测性基础设施，包括健康检查、API 限流、全局异常处理和请求日志。

### 1.3 验收标准

| # | 验收条件 |
|---|---------|
| AC1 | `/actuator/health` 返回各组件（DB/Redis）健康状态 |
| AC2 | `/actuator/metrics` 暴露 Prometheus 指标 |
| AC3 | API 限流：短时间内大量请求 → 429 Too Many Requests |
| AC4 | 任何未捕获异常 → 统一 JSON 错误响应 |
| AC5 | 每个请求记录：method、path、status、耗时 |

## 二、开发方案

### 2.1 技术方案

| 技术点 | 选型 | 说明 |
|--------|------|------|
| 健康检查 | Spring Actuator | 内置 DB/Redis 健康指示器 |
| 指标暴露 | Micrometer + Prometheus | `/actuator/prometheus` |
| API 限流 | Bucket4j | 令牌桶算法，基于 IP |
| 异常处理 | `@RestControllerAdvice` | 统一 JSON 错误响应 |
| 请求日志 | `Filter` 或 `Interceptor` | 记录 method + path + status + duration |

### 2.2 关键文件

| 文件 | 说明 |
|------|------|
| `backend/.../config/GlobalExceptionHandler.java` | 全局异常处理 |
| `backend/.../config/RequestLoggingFilter.java` | 请求日志拦截器 |
| `backend/.../config/RateLimitFilter.java` | API 限流过滤器 |
| `backend/src/main/resources/application.yml` | Actuator + Micrometer 配置 |

**步骤 2：全局异常处理**

```java
@RestControllerAdvice
public class GlobalExceptionHandler {

    @ExceptionHandler(MethodArgumentNotValidException.class)
    public ResponseEntity<ApiError> handleValidation(MethodArgumentNotValidException ex) {
        String message = ex.getBindingResult().getFieldErrors().stream()
            .map(e -> e.getField() + ": " + e.getDefaultMessage())
            .collect(Collectors.joining("; "));
        return ResponseEntity.badRequest()
            .body(ApiError.of(400, message));
    }

    @ExceptionHandler(AuthenticationException.class)
    public ResponseEntity<ApiError> handleAuth(AuthenticationException ex) {
        return ResponseEntity.status(401)
            .body(ApiError.of(401, "认证失败，请重新登录"));
    }

    @ExceptionHandler(EntityNotFoundException.class)
    public ResponseEntity<ApiError> handleNotFound(EntityNotFoundException ex) {
        return ResponseEntity.status(404)
            .body(ApiError.of(404, ex.getMessage()));
    }

    @ExceptionHandler(Exception.class)
    public ResponseEntity<ApiError> handleGeneral(Exception ex) {
        log.error("Unhandled exception", ex);
        return ResponseEntity.status(500)
            .body(ApiError.of(500, "服务器内部错误，请稍后重试"));
    }

    static record ApiError(int code, String message, long timestamp) {
        static ApiError of(int code, String message) {
            return new ApiError(code, message, System.currentTimeMillis());
        }
    }
}
```

**步骤 3：请求日志过滤器**

```java
@Component
public class RequestLoggingFilter extends OncePerRequestFilter {
    private static final Logger log = LoggerFactory.getLogger("ACCESS");

    @Override
    protected void doFilterInternal(HttpServletRequest request,
            HttpServletResponse response, FilterChain chain)
            throws ServletException, IOException {
        long start = System.currentTimeMillis();
        String method = request.getMethod();
        String path = request.getRequestURI();
        String query = request.getQueryString();
        String clientIp = request.getRemoteAddr();

        try {
            chain.doFilter(request, response);
        } finally {
            long duration = System.currentTimeMillis() - start;
            int status = response.getStatus();
            log.info("{} {} [{}] {} {}ms", method,
                query != null ? path + "?" + query : path,
                clientIp, status, duration);
        }
    }
}
```

**步骤 4：API 限流过滤器**

```java
@Component
public class RateLimitFilter extends OncePerRequestFilter {
    private final ConcurrentHashMap<String, Bucket> buckets = new ConcurrentHashMap<>();

    // 每秒 20 个请求
    private Bucket createBucket() {
        return Bucket4j.builder()
            .addLimit(Bandwidth.classic(20, Refill.greedy(20, Duration.ofSeconds(1))))
            .build();
    }

    @Override
    protected void doFilterInternal(HttpServletRequest request,
            HttpServletResponse response, FilterChain chain)
            throws ServletException, IOException {

        String key = request.getRemoteAddr();
        Bucket bucket = buckets.computeIfAbsent(key, k -> createBucket());

        if (bucket.tryConsume(1)) {
            chain.doFilter(request, response);
        } else {
            response.setStatus(429);
            response.setContentType("application/json");
            response.getWriter().write("{\"code\":429,\"message\":\"请求过于频繁，请稍后再试\"}");
        }
    }
}
```
