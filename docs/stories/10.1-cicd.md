# Story 10.1 — CI/CD 与部署

> **Epic**: infra (系统基建) | **优先级**: P3 | **阶段**: Phase 4 商业化
> **依赖**: 无（贯穿全阶段） | **预估工时**: 2d

---

## 一、PRD（产品需求文档）

### 1.1 背景

系统需要标准化的容器化部署方案和 CI/CD 流水线，确保开发、测试、生产环境的一致性和自动化。

### 1.2 需求描述

作为团队，我们需要自动化构建、测试和部署流程，支持一键部署和停止。

### 1.3 验收标准

| # | 验收条件 |
|---|---------|
| AC1 | `docker-compose up -d` → 8 个服务全部启动成功 |
| AC2 | `curl localhost:3000` → 前端页面正常 |
| AC3 | `curl localhost:8080/api/v1/health` → `{"status":"UP"}` |
| AC4 | `curl localhost:8000/ai/v1/health` → `{"status":"OK"}` |
| AC5 | GitHub Actions CI: 提交代码 → 自动构建+测试 |
| AC6 | `deploy.sh` 一键部署脚本可用 |

---

## 二、开发方案

### 2.1 技术方案

| 技术点 | 选型 | 说明 |
|--------|------|------|
| 容器化 | Docker + Docker Compose | 8 个服务统一编排 |
| Java 镜像 | 多阶段构建 | maven:3.9-eclipse-temurin-17 → eclipse-temurin:17-jre |
| Python 镜像 | python:3.11-slim | 精简镜像 |
| 前端镜像 | node:18-alpine → nginx:alpine | 多阶段构建 |
| CI/CD | GitHub Actions | 自动 test + build |
| 反向代理 | Nginx | 前端静态资源 + API 代理 |

### 2.2 关键文件

| 文件 | 说明 |
|------|------|
| `devops/docker-compose.yml` | 8 服务编排 |
| `devops/Dockerfile` | Spring Boot 镜像 |
| `devops/AIServer.Dockerfile` | Python FastAPI 镜像 |
| `devops/nginx/Dockerfile` | Vue 3 + Nginx 镜像 |
| `devops/nginx/default.conf` | Nginx 反向代理配置 |
| `devops/scripts/deploy.sh` | 一键部署 |
| `devops/scripts/stop.sh` | 停止服务 |
| `devops/.env` | 环境变量模板 |
| `.github/workflows/ci.yml` | CI 流水线 |

### 2.3 实施步骤

**步骤 1：docker-compose.yml**

```yaml
version: '3.8'
services:
  frontend:
    build:
      context: ../frontend
      dockerfile: ../devops/nginx/Dockerfile
    ports: ["3000:80"]
    depends_on: [backend]
    networks: [smartreport-net]

  backend:
    build:
      context: ../backend
      dockerfile: ../devops/Dockerfile
    ports: ["8080:8080"]
    environment:
      SPRING_DATASOURCE_URL: jdbc:mysql://mysql:3306/smartreport?useUnicode=true&characterEncoding=utf8mb4
      SPRING_DATASOURCE_USERNAME: ${MYSQL_USER:-root}
      SPRING_DATASOURCE_PASSWORD: ${MYSQL_PASSWORD:-root123}
      SPRING_REDIS_HOST: redis
      AI_ENGINE_URL: http://ai-engine:8000
      MINIO_ENDPOINT: http://minio:9000
    depends_on: [mysql, redis]
    networks: [smartreport-net]

  ai-engine:
    build:
      context: ../ai-engine
      dockerfile: ../devops/AIServer.Dockerfile
    ports: ["8000:8000"]
    environment:
      OPENAI_API_KEY: ${OPENAI_API_KEY}
      DEEPSEEK_API_KEY: ${DEEPSEEK_API_KEY}
    depends_on: [rocketmq-broker]
    networks: [smartreport-net]

  mysql:
    image: mysql:8.0
    ports: ["3306:3306"]
    environment:
      MYSQL_ROOT_PASSWORD: ${MYSQL_PASSWORD:-root123}
      MYSQL_DATABASE: smartreport
    volumes:
      - ./mysql/sql:/docker-entrypoint-initdb.d
      - mysql-data:/var/lib/mysql
    networks: [smartreport-net]

  redis:
    image: redis:7-alpine
    ports: ["6379:6379"]
    volumes: [redis-data:/data]
    networks: [smartreport-net]

  rocketmq-namesrv:
    image: apache/rocketmq:5.1.0
    command: sh mqnamesrv
    ports: ["9876:9876"]
    networks: [smartreport-net]

  rocketmq-broker:
    image: apache/rocketmq:5.1.0
    command: sh mqbroker -n rocketmq-namesrv:9876
    ports: ["10911:10911"]
    depends_on: [rocketmq-namesrv]
    networks: [smartreport-net]

  minio:
    image: minio/minio
    command: server /data --console-address ":9001"
    ports: ["9000:9000", "9001:9001"]
    environment:
      MINIO_ROOT_USER: minioadmin
      MINIO_ROOT_PASSWORD: minioadmin
    volumes: [minio-data:/data]
    networks: [smartreport-net]

volumes:
  mysql-data: redis-data: minio-data:

networks:
  smartreport-net: driver: bridge
```

**步骤 2：Dockerfile（Spring Boot）**

```dockerfile
# devops/Dockerfile
FROM maven:3.9-eclipse-temurin-17 AS build
WORKDIR /app
COPY pom.xml .
COPY src ./src
RUN mvn package -DskipTests -q

FROM eclipse-temurin:17-jre
WORKDIR /app
COPY --from=build /app/target/*.jar app.jar
EXPOSE 8080
ENTRYPOINT ["java", "-jar", "app.jar"]
```

**步骤 3：AIServer.Dockerfile**

```dockerfile
# devops/AIServer.Dockerfile
FROM python:3.11-slim
WORKDIR /app
COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt
COPY app ./app
EXPOSE 8000
CMD ["uvicorn", "app.main:app", "--host", "0.0.0.0", "--port", "8000"]
```

**步骤 4：Nginx 配置**

```nginx
# devops/nginx/default.conf
server {
    listen 80;
    server_name localhost;

    # 前端静态资源
    location / {
        root /usr/share/nginx/html;
        index index.html;
        try_files $uri $uri/ /index.html;
    }

    # API 代理到 Java 后端
    location /api/ {
        proxy_pass http://backend:8080;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }

    # SSE 长连接支持
    location /api/v1/chat/ {
        proxy_pass http://backend:8080;
        proxy_set_header Host $host;
        proxy_buffering off;
        proxy_cache off;
        proxy_read_timeout 300s;
    }
}
```

**步骤 5：GitHub Actions CI**

```yaml
# .github/workflows/ci.yml
name: CI

on:
  push:
    branches: [main, develop]
  pull_request:
    branches: [main]

jobs:
  frontend:
    runs-on: ubuntu-latest
    defaults: { run: { working-directory: frontend } }
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with: { node-version: 18 }
      - run: npm ci
      - run: npm run lint
      - run: npm run build

  backend:
    runs-on: ubuntu-latest
    defaults: { run: { working-directory: backend } }
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-java@v4
        with: { java-version: 17, distribution: temurin }
      - run: mvn test
      - run: mvn package -DskipTests

  ai-engine:
    runs-on: ubuntu-latest
    defaults: { run: { working-directory: ai-engine } }
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-python@v5
        with: { python-version: '3.11' }
      - run: pip install -r requirements.txt
      - run: pytest
      - run: ruff check .
```

**步骤 6：deploy.sh / stop.sh**

```bash
#!/bin/bash
# devops/scripts/deploy.sh
set -e
cd "$(dirname "$0")/.."
docker-compose down
docker-compose build
docker-compose up -d
echo "✅ SmartReport 部署完成"
echo "前端: http://localhost:3000"
echo "后端: http://localhost:8080"
echo "MinIO: http://localhost:9001"
```

```bash
#!/bin/bash
# devops/scripts/stop.sh
cd "$(dirname "$0")/.."
docker-compose down
echo "✅ 所有服务已停止"
```
