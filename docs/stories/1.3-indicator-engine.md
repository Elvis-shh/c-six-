# Story 1.3 — 财报指标计算引擎

> **Epic**: data-engine (财报数据引擎) | **优先级**: P0 | **阶段**: Phase 1 MVP
> **依赖**: Story 1.1 | **预估工时**: 2d

---

## 一、PRD（产品需求文档）

### 1.1 背景

原始财报数据入库后，需要计算各类衍生指标（同比、环比、复合增长率、财务比率）以支撑前端 KPI 卡片、趋势图表和分析模块。

### 1.2 需求描述

作为系统，我需要根据原始财报数据自动计算衍生指标（同比、环比、毛利率、净利率、ROE 等），所有计算在后端完成，前端不再做财务公式运算。

### 1.3 验收标准

| # | 验收条件 | 验证方式 |
|---|---------|---------|
| AC1 | `calculateYoY(150, 120)` 返回 `25.0`（同比增长 25%） | 单元测试 |
| AC2 | `calculateYoY(0, 0)` 返回 `null`（除零处理） | 单元测试 |
| AC3 | `calculateCAGR([100,120,150], 3)` 返回复合增长率 ~22.47% | 单元测试 |
| AC4 | `calculateRatio(80, 100)` 返回 `80.0` | 单元测试 |
| AC5 | 负数输入正常计算（如亏损同比收窄） | 单元测试 |
| AC6 | 所有计算逻辑单元测试覆盖率 > 90% | `mvn test` jacoco |

### 1.4 计算类型定义

| 计算类型 | 公式 | 适用指标 | 返回值 |
|---------|------|---------|--------|
| YoY（同比） | `(current - prev) / |prev| * 100` | 营收、净利润、现金流 | `BigDecimal` |
| QoQ（环比） | `(current - prev) / |prev| * 100` | 季度数据 | `BigDecimal` |
| CAGR（复合增长率） | `(end/start)^(1/n) - 1` | 多年趋势 | `BigDecimal` |
| Ratio（比率） | `numerator / denominator * 100` | 毛利率、净利率、ROE | `BigDecimal` |

## 二、开发方案

### 2.1 技术方案

| 技术点 | 选型 | 说明 |
|--------|------|------|
| 精度 | `BigDecimal` | 财务计算禁止浮点数，避免舍入误差 |
| 边界处理 | 返回 `null` | 除零、空值不抛异常，返回 null 由上层处理 |
| 舍入模式 | `RoundingMode.HALF_UP` | 四舍五入，保留 2 位小数 |
| 测试 | JUnit 5 + AssertJ | 参数化测试覆盖边界值 |

### 2.2 关键文件

| 文件 | 说明 |
|------|------|
| `backend/.../service/IndicatorService.java` | 接口定义 |
| `backend/.../service/IndicatorServiceImpl.java` | 计算实现 |
| `backend/src/test/.../service/IndicatorServiceTest.java` | 单元测试 |

### 2.3 实施步骤

**步骤 1：定义 IndicatorService 接口**

```java
public interface IndicatorService {
    /**
     * 计算同比增长率
     * @return 百分比值，如 25.0 表示 25%；除零返回 null
     */
    BigDecimal calculateYoY(BigDecimal current, BigDecimal previous);

    /**
     * 计算环比增长率
     */
    BigDecimal calculateQoQ(BigDecimal current, BigDecimal previous);

    /**
     * 计算复合年增长率
     * @param values 按时间顺序排列的数值
     * @param years 年数
     */
    BigDecimal calculateCAGR(List<BigDecimal> values, int years);

    /**
     * 计算比率
     * @return 百分比值；分母为零返回 null
     */
    BigDecimal calculateRatio(BigDecimal numerator, BigDecimal denominator);
}
```

**步骤 2：实现 IndicatorServiceImpl**

关键逻辑：
- YoY: `if (prev == null || prev.compareTo(ZERO) == 0) return null;` → `current.subtract(prev).divide(prev.abs(), MATH_CONTEXT).multiply(HUNDRED)`
- CAGR: `Math.pow(end/start, 1.0/n) - 1`，注意 BigDecimal 转 double 精度损失
- Ratio: 分母为 0 返回 null

**步骤 3：编写单元测试**

使用 `@ParameterizedTest` + `@CsvSource` 覆盖：

| 测试场景 | 输入 | 预期 |
|---------|------|------|
| 正常同比 | current=150, prev=120 | 25.00 |
| 同比下降 | current=80, prev=100 | -20.00 |
| 除零 | current=100, prev=0 | null |
| 亏损同比 | current=-50, prev=-100 | 50.00（收窄） |
| 转盈 | current=100, prev=-50 | 300.00 |
| null 输入 | current=null | null |
| CAGR 3年 | [100, 120, 144] | ~20.00 |
| Ratio 正常 | num=80, den=100 | 80.00 |
| Ratio 除零 | num=100, den=0 | null |

### 2.4 测试用例
