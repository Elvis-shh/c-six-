# Story 3.1 — KPI 指标卡片

> **Epic**: dashboard (财报总览看板) | **优先级**: P1 | **阶段**: Phase 1 MVP
> **依赖**: Story 2.2 | **预估工时**: 2d

## 一、PRD（产品需求文档）

### 1.1 背景

用户进入 Dashboard 后，第一眼需要看到公司最核心的 4 个关键指标（营收、净利润、资产负债率、现金流）及同比变化趋势。KPI 卡片是财报总览的"第一印象"。

### 1.2 需求描述

作为用户，我可以在页面顶部看到 4 个核心 KPI（营收、净利润、资产负债率、现金流）及同比变化。

### 1.3 验收标准

| # | 验收条件 |
|---|---------|
| AC1 | 页面顶部渲染 4 张 KPI 卡片，2×2 网格布局（移动端 1 列） |
| AC2 | 每张卡片显示：指标名称、数值（含单位）、同比变化率 |
| AC3 | 数值有 count-up 动画效果（从 0 滚动到目标值） |
| AC4 | 同比 ↑↓ 箭头颜色：增长=绿色、下降=红色（资产负债率反向） |
| AC5 | 卡片切换公司时，数值重新执行 count-up 动画 |
| AC6 | 点击卡片可跳转到对应指标的详细图表区域 |

### 1.4 UI 设计

```
┌──────────────────────┐  ┌──────────────────────┐
│ 营业总收入           │  │ 归母净利润           │
│ 1,748 亿            │  │ 862 亿               │
│ ↑ 16.1%（绿色）     │  │ ↑ 15.4%（绿色）      │
└──────────────────────┘  └──────────────────────┘
┌──────────────────────┐  ┌──────────────────────┐
│ 资产负债率           │  │ 经营现金流           │
│ 16.1%               │  │ 810 亿               │
│ ↓ 1.1%（绿色，好）  │  │ ↑ 21.6%（绿色）      │
└──────────────────────┘  └──────────────────────┘
```

**颜色规则**（金融行业惯例）：
- 营收/净利润/现金流 ↑ → 绿色（涨=好）
- 营收/净利润/现金流 ↓ → 红色（跌=坏）
- 资产负债率 ↑ → 红色（负债率上升=坏）
- 资产负债率 ↓ → 绿色（负债率下降=好）

### 2.3 实施步骤

**步骤 1：后端 KPI 端点**

```java
@GetMapping("/{companyCode}/kpi")
public ApiResponse<KpiResponse> getKpi(@PathVariable String companyCode) {
    FinancialReport latest = reportService.getLatestReport(companyCode);
    FinancialReport previous = reportService.getPreviousYearReport(companyCode, latest.getReportYear());

    return ApiResponse.success(KpiResponse.builder()
        .company(CompanyBrief.from(latest.getCompany()))
        .reportYear(latest.getReportYear())
        .reportType(latest.getReportType())
        .kpis(List.of(
            buildKpi("revenue", "营业总收入", latest, previous),
            buildKpi("profit", "归母净利润", latest, previous),
            buildKpi("debtRatio", "资产负债率", latest, previous),
            buildKpi("cashFlow", "经营现金流", latest, previous)
        ))
        .build());
}

private KpiItem buildKpi(String key, String name, FinancialReport current, FinancialReport prev) {
    BigDecimal value = indicatorService.getValue(current, key);
    BigDecimal prevValue = indicatorService.getValue(prev, key);
    BigDecimal yoy = indicatorService.calculateYoY(value, prevValue);
    String trend = determineTrend(key, yoy);  // up/down/down_good
    return new KpiItem(key, name, value, indicatorService.getUnit(key), yoy, trend);
}
```

**步骤 2：实现 count-up 动画**

```typescript
// composables/useCountUp.ts
export function useCountUp(target: Ref<number>, duration = 1000) {
  const display = ref(0)

  watch(target, (newVal) => {
    const startVal = display.value
    const startTime = performance.now()
    const diff = newVal - startVal

    function animate(now: number) {
      const elapsed = now - startTime
      const progress = Math.min(elapsed / duration, 1)
      // easeOutQuad 缓动
      const eased = 1 - (1 - progress) * (1 - progress)
      display.value = startVal + diff * eased
      if (progress < 1) requestAnimationFrame(animate)
    }
    requestAnimationFrame(animate)
  }, { immediate: true })

  return { display }
}
```

**步骤 3：实现 KpiCards.vue**

```vue

      :value="kpi.value"
      :unit="kpi.unit"
      :yoy="kpi.yoy"
      :trend="kpi.trend"
      @click="scrollTo(kpi.key)"
    />
  </div>
</template>
```

**步骤 4：实现 KpiCard.vue**

Props:
- `name`: 指标名称
- `value`: 数值
- `unit`: 单位（亿/%）
- `yoy`: 同比变化率（可为 null）
- `trend`: `'up' | 'down' | 'down_good'`

显示逻辑：
- `trend === 'up'` → 绿色 ↑ + yoy%
- `trend === 'down'` → 红色 ↓ + yoy%
- `trend === 'down_good'` → 绿色 ↓ + yoy%（资产负债率下降）
- `yoy === null` → 灰色 "—"

格式化：
- 营收/利润/现金流 → Big Number 格式（如 1,748 亿）
- 资产负债率 → 百分比（如 16.1%）
