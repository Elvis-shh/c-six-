## 一、概述

### 1.1 项目背景

SmartReport 是一款智能财报研读与分析助手，面向 A 股投资者，提供搜索公司 → 财报 KPI 看板 → 亮点风险分析 → 趋势预测 → AI 智能问答的完整分析链路。

### 1.2 测试目标

- 验证所有功能模块符合验收标准（AC）
- 发现并跟踪缺陷，确保核心链路稳定可用
- 评估 AI 引擎（LLM / RAG / OCR / NER）的质量与准确性
- 验证前后端集成及 Docker 部署的可靠性
- 为后续用户体验调查问卷提供功能验证基础

### 1.3 测试范围

| 层级 | 范围 |
|------|------|
| **前端** | Vue 3 页面渲染、组件交互、路由跳转、导出功能、响应式布局 |
| **后端** | Spring Boot API 正确性、JPA 数据访问、规则引擎、异步任务 |
| **AI 引擎** | LLM 对话质量、RAG 检索准确性、NER 提取精度、OCR 识别（含 mock） |
| **数据库** | 17 张表结构校验、种子数据完整性、索引有效性 |
| **集成** | 前后端联调、Java ↔ Python 通信、Docker Compose 一键部署 |
| **非功能** | 性能（响应时间）、安全性（JWT 待实施）、兼容性（移动端） |

### 1.4 不在范围

- 用户系统（Epic 9）— 尚未实施，仅做接口预留验证
- CI/CD 与监控（Epic 10）— 尚未实施
- 真实 OCR 图片识别 — AI 引擎目前为非 txt 文件返回 mock 数据
- 服务端真实 PDF/Word/Excel 导出 — 后端导出为占位 HTML

## 二、测试环境

### 2.1 环境配置

| 项目 | 配置 |
|------|------|
| **部署方式** | Docker Compose（`devops/docker-compose.yml`） |
| **前端地址** | `http://localhost:3000` |
| **后端地址** | `http://localhost:8080` |
| **AI 引擎地址** | `http://localhost:8000` |
| **数据库** | MySQL 8.0 `localhost:3307`，库名 `smartreport` |
| **LLM Provider** | 默认 `mock`（`.env` 中 `LLM_PROVIDER=mock`） |
| **浏览器** | Chrome / Edge 最新版，移动端模拟（DevTools） |

### 2.2 启动命令

```bash
cd devops
docker compose --env-file .env up -d --build
```

| MySQL | `docker exec smartreport-mysql mysql -u smartreport -psmartreport123 -e "SHOW TABLES"` |

## 三、测试策略

### 3.1 测试分层

```
┌─────────────────────────────────────────────────┐
│              手动探索测试（E2E）                  │
│        覆盖：核心用户旅程、跨模块集成              │

├─────────────────────────────────────────────────┤
│            单元测试（已有 + 待补充）               │
│    覆盖：指标计算、规则引擎、正则提取              │
└─────────────────────────────────────────────────┘
```

### 3.2 测试优先级

| 优先级 | 模块 | 原因 |
|--------|------|------|
| **P0（必测）** | 数据引擎、搜索、页面跳转、KPI 看板 | 核心 MVP 闭环 |
| **P1（重点）** | 亮点风险、预测、聊天、导出 | 核心差异化功能 |
| **P2（选测）** | 文件上传、OCR/NER、行业对比 | AI 增强功能 |
| **P3（预留）** | 用户系统、CI/CD、监控 | 尚未实施 |

  ├─ 行业均值与排名（TC-1.4-*）

### 7.2 缺陷管理

- **严重**（Blocker）：核心链路不可用，P0 用例不通过 → 必须立即修复
- **高**（Critical）：P1 功能异常，影响主要用户体验 → 本迭代修复
- **中**（Major）：P2 功能异常或 UI 瑕疵 → 下迭代修复
- **低**（Minor）：优化建议、文案调整 → 记录 backlog

### 7.3 通过标准

- P0 用例通过率 ≥ **100%**
- P1 用例通过率 ≥ **95%**
- P2 用例通过率 ≥ **80%**
- 无 Blocker 级别缺陷
- 核心 E2E 旅程全部通过
