# SmartReport 用户体验调查问卷 — 设计方案

> 版本：1.0  
> 日期：2026-06-22  
> 用途：面向真实用户收集产品使用体验反馈

---

## 一、问卷目标

| 维度 | 目标 |
|------|------|
| **功能可用性** | 用户能否顺利完成核心任务（搜索→看板→分析） |
| **信息可读性** | KPI/表格/图表的清晰度和理解难度 |
| **分析价值感** | 亮点/风险/预测是否让用户觉得"有用" |
| **AI 信任度** | 智能问答的回答质量、准确性、引用可信度 |
| **操作效率** | 完成任务所需的步骤和时间感受 |
| **整体满意度** | NPS 净推荐值、继续使用意愿、付费意愿 |
| **改进方向** | 用户最希望增加/优化的功能 |

## 二、问卷结构（共 6 部分，25 题）

### Part A：用户画像（3 题）— 单选

| # | 问题 | 选项 |
|---|------|------|
| A1 | 您的投资经验是？ | ① 入门（<1年）② 有一定经验（1-3年）③ 较丰富（3-10年）④ 专业（>10年） |
| A2 | 您平时阅读财报的频率？ | ① 几乎不看 ② 偶尔看（季度/半年）③ 经常看（每月）④ 频繁（每周） |

## 三、实施计划

### 3.1 问卷形式

| 方式 | 工具 | 适用场景 |
|------|------|----------|
| **线上问卷** | 腾讯问卷 / 问卷星 / Google Forms | 远程分发，覆盖面广 |
| **现场访谈** | 面对面 + 录屏 | 深度观察操作行为 |
| **嵌入产品** | 应用内弹窗 | 使用后即时反馈 |

> **建议**：先用**线上问卷**收集 30-50 份定量数据，再挑选 3-5 位用户做**现场访谈**获取定性洞察。

### 3.2 用户招募

| 画像 | 目标数量 | 招募渠道 |
|------|:--------:|----------|
| 个人投资者（入门） | 10-15 人 | 雪球/微博/投资群 |
| 个人投资者（资深） | 10-15 人 | 同上 |
| 财经从业者 | 5-10 人 | 朋友圈/行业社群 |
| 学生/学习者 | 5-10 人 | 高校财经社团 |

### 3.3 执行时间线

```
第 1 天：部署测试环境，准备问卷链接和测试账号
第 2-3 天：内部预测试（2-3 人试填，修正问题措辞）
第 4-10 天：正式发放问卷（目标 30-50 份）
第 11-12 天：数据分析 + 产出报告
```

### 3.4 用户任务引导

用户在填写问卷前，需完成以下核心任务（预计 10-15 分钟）：

```
任务 1：搜索"贵州茅台"→ 查看 KPI 卡片和5年数据表 → 切换折线图
任务 2：查看经营亮点和风险提示 → 切换到趋势预测
任务 3：打开AI聊天面板 → 提问"这家公司的盈利能力怎么样？"
任务 4：导出PNG报告 → 切换另一家公司（自行选择）
```

## 四、数据分析方法

| 指标 | 计算方式 | 目标阈值 |
|------|----------|:--------:|
| **NPS** | 推荐者(9-10)占比 - 贬损者(0-6)占比 | > 0（及格）> 30（优秀） |
| **满意度均分** | 各量表题 1-5 分均值 | ≥ 3.5 |
| **任务完成率** | 成功完成4个任务的比例 | ≥ 85% |
| **痛点词云** | 开放题高频词提取 | — |

### 交叉分析维度

- 入门用户 vs 资深用户的体验差异
- 常看财报 vs 不常看财报的用户对AI回答的信任度差异
- NPS 高低分组的关键差异点
